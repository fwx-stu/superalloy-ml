# 常见问题

## 现在使用的是真实数据吗？

默认不是。

项目默认使用合成高温合金表格数据，目的是让流程可以直接运行。

## 可以替换成自己的数据吗？

可以。

支持 CSV 和 Excel 表格数据。

推荐放在：

```text
data/raw/
```

例如：

```text
data/raw/my_superalloy_data.csv
data/raw/my_superalloy_data.xlsx
```

## 替换数据时需要哪些字段？

至少需要：

```text
元素成分字段
工艺字段
测试条件字段
目标列
```

如果只做某一个目标预测，只需要保留对应目标列。

例如只预测蠕变寿命，需要包含：

```text
creep_log_life_h
```

## 元素成分总和必须等于 100 吗？

默认会检查元素总和是否为 100。

如果原始数据中元素成分没有归一化，可以在特征工程中归一化。

如果只是想跳过检查，可以使用脚本参数：

```bash
python scripts/run_data_check.py --allow-composition-not-100
```

## 为什么有些模型需要标准化？

KNN、SVM、SVR、线性模型对特征尺度敏感。

项目默认使用 sklearn 的 Pipeline，把预处理和模型训练放在一起，减少数据泄漏风险。

## 为什么树模型也放在 Pipeline 里？

统一建模流程，方便交叉验证、保存模型和替换算法。

虽然树模型通常不依赖标准化，但统一 Pipeline 更方便维护。

## 为什么 SHAP 可能运行失败？

SHAP 是可选依赖，可能因为环境、模型类型或版本问题失败。

脚本中已经做了兼容处理，SHAP 失败不会影响其他解释性分析结果。

## 深度学习部分为什么单独安装？

PyTorch 体积较大，不放在基础依赖中。

需要运行深度学习部分时，再安装：

```bash
pip install -r requirements-dl.txt
```

## 运行结果在哪里？

统一保存在：

```text
outputs/
```

其中：

```text
outputs/models/       模型文件
outputs/reports/      结果表和报告
outputs/figures/      图像
outputs/predictions/  预测结果
```

## 可以只运行某一部分吗？

可以。

每个模块都有独立脚本，例如：

```bash
python scripts/run_traditional_ml.py
python scripts/run_clustering.py
python scripts/run_dimensionality_reduction.py
python scripts/run_shap_analysis.py
```
